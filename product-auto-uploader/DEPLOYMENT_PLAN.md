# 4인 배포용 운영 계획

## 1. 목표

이 프로젝트는 개발자가 직접 쓰는 스크립트가 아니라, 4명의 사용자가 각자 PC에서 실행하는 로컬 도구로 배포한다.

따라서 우선순위는 아래 순서로 잡는다.

1. 설치가 쉬워야 한다.
2. 매번 로그인 정보를 다시 넣지 않아야 한다.
3. 실패 시 원인을 추적할 수 있어야 한다.
4. 사이트 셀렉터 변경 시 전체 구조를 갈아엎지 않아야 한다.

## 2. 로그인 정보 저장 정책

권장안은 `아이디/비밀번호 JSON 저장`이 아니라 `브라우저 세션 저장`이다.

### 저장하지 않는 것

- 아이디
- 비밀번호
- 민감한 인증 정보

### 저장하는 것

- 사용자 설정 파일
- 마지막 입력값
- Playwright 브라우저 프로필

이 방식의 장점:

- 사용자 편의성 확보
- 비밀번호 평문 저장 회피
- 사이트 로그인 플로우 변경에도 비교적 대응 가능

## 3. 사용자별 저장 위치

각 사용자 PC에서 아래 위치를 사용한다.

- `%APPDATA%\ProductAutoUploader\user-config.json`
- `%LOCALAPPDATA%\ProductAutoUploader\playwright-profile`
- `%LOCALAPPDATA%\ProductAutoUploader\logs`
- `%LOCALAPPDATA%\ProductAutoUploader\screenshots`
- `%LOCALAPPDATA%\ProductAutoUploader\output`

이렇게 두면 동일한 배포본을 여러 사용자에게 전달해도 설정 충돌이 줄어든다.

## 4. 배포 형태

1차는 아래 형태가 가장 현실적이다.

- 압축 파일 배포
- `setup.bat`
- `run_gui.bat`
- 짧은 사용법 문서

2차는 필요 시 `PyInstaller`로 exe 패키징한다.

## 5. 사용자 흐름

### 최초 1회

1. 설치
2. 앱 실행
3. 이미지 루트 폴더 입력
4. 머스트잇 등록 URL 입력
5. 로그인 확인 셀렉터 입력
6. 로그인 세션 준비

### 이후 반복 사용

1. 앱 실행
2. 상품 정보 입력
3. 미리보기 실행
4. 필요 시 최종 제출

## 6. 운영상 주의점

- 로그인 세션이 만료되면 다시 `로그인 세션 준비`를 실행한다.
- 셀렉터가 바뀌면 `selectors/mustit.json`을 수정한다.
- 초기에는 `preview` 모드만 기본 사용한다.
- 자동 제출은 충분히 검증된 뒤 일부 사용자에게만 연다.
